1. EXECUTIVE SUMMARY

OBIETTIVO DEL PROGETTO:

Creare una piattaforma no-code che permetta a squadre, associazioni e club sportivi
(amatoriali, giovanili, dilettantistici) di generare la propria app personalizzata
per gestire la comunicazione con tifosi, soci, genitori e membri.

DIFFERENZA CHIAVE rispetto ai giochi manageriali:
• Non è un gioco, è uno strumento REALE per club REALI
• Focus su COMUNICAZIONE e COINVOLGIMENTO, non su simulazione
• Target: club piccoli e medi che vogliono professionalità digitale
• Obiettivo: far sentire tifosi più vicini alla squadra

TARGET UTENTE:
• Squadre di calcio dilettantistiche (Serie D, Eccellenza, Promozione)
• Club giovanili (settorini giovanili)
• Società sportive dilettantistiche (basket, volley, rugby)
• Associazioni sportive culturali
• Team amatoriali

VALORE PROPOSTO:
"Dai al tuo club l'app che si merita, senza costi di sviluppo"

FUNZIONALITÀ CHIAVE:
✓ Live scoring durante le partite
✓ Notifiche push in tempo reale
✓ Calendario eventi sincronizzato
✓ Gestione iscrizioni e pagamenti
✓ Area foto/video condivisa
✓ Visibilità sponsor
✓ Chat di squadra
✓ Statistiche giocatori
✓ Classifiche e risultati

3. FUNZIONALITÀ CORE PER CLUB SPORTIVI REALI
3.1 Gestione Membri e Iscrizioni

FUNZIONALITÀ ESSENZIALI:

1. REGISTRAZIONE ONLINE:
  • Form personalizzabili per raccolta dati
  • Upload documenti (certificati medici, liberatorie)
  • Firma digitale per waiver e consensi
  • Registrazione familiare (genitori + figli)

2. DATABASE MEMBRI:
  • Profili completi con foto
  • Categorie membri (giocatore, genitore, volontario, staff)
  • Stato iscrizione (attivo, in attesa, scaduto)
  • Storico partecipazione

3. MEMBERSHIP TIERS:
  • Tipologie iscrizione (annuale, semestrale, trimestrale)
  • Livelli membership (base, premium, sostenitore)
  • Benefits per livello

4. DIGITAL MEMBERSHIP CARDS:
  • Card con QR code per check-in
  • Wallet integration (Apple/Google Pay)
  • Validazione automatica

5. RINNOVI AUTOMATICI:
  • Reminder via email/push
  • Auto-renewal con pagamento
  • Tracking scadenze

5. SCHEMA DATABASE PER CLUB SPORTIVI

Schema database ottimizzato per club sportivi reali con focus su:
• Gestione membri e iscrizioni
• Comunicazione e notifiche
• Eventi e calendario
• Live scoring
• Sponsor visibility

5.1 Tabelle Core

-- ============================================
-- CLUBS
-- ============================================
CREATE TABLE clubs (
   id              SERIAL PRIMARY KEY,
   name            VARCHAR(200) NOT NULL,
   slug            VARCHAR(100) UNIQUE,
   sport_type      VARCHAR(50) NOT NULL,

   -- Info
   description     TEXT,
   founded_year    INTEGER,
   colors_primary  VARCHAR(7),
   colors_secondary VARCHAR(7),

   -- Contact
   email           VARCHAR(200),
   phone           VARCHAR(50),
   website         VARCHAR(500),

   -- Location
   address         TEXT,
   city            VARCHAR(100),
   country         VARCHAR(3) DEFAULT 'ITA',

   -- Settings
   logo_url        VARCHAR(500),
   banner_url      VARCHAR(500),

   -- Status
   is_active       BOOLEAN DEFAULT true,
   subscription_plan VARCHAR(50) DEFAULT 'free',

   created_at      TIMESTAMP DEFAULT NOW(),
   updated_at      TIMESTAMP DEFAULT NOW()
);

-- ============================================
-- USERS (Members)
-- ============================================
CREATE TABLE users (
   id              SERIAL PRIMARY KEY,
   club_id         INTEGER REFERENCES clubs(id),

   -- Auth
   email           VARCHAR(200) UNIQUE,
   password_hash   VARCHAR(255),

   -- Profile
   first_name      VARCHAR(100),
   last_name       VARCHAR(100),
   display_name    VARCHAR(200),
   phone           VARCHAR(50),
   birth_date      DATE,

   -- Role
   role            VARCHAR(50) DEFAULT 'member', -- admin, coach, player, parent, member

   -- Status
   is_active       BOOLEAN DEFAULT true,
   email_verified  BOOLEAN DEFAULT false,

   -- Media
   avatar_url      VARCHAR(500),

   created_at      TIMESTAMP DEFAULT NOW(),
   updated_at      TIMESTAMP DEFAULT NOW()
);

-- ============================================
-- TEAMS (Squadre all'interno del club)
-- ============================================
CREATE TABLE teams (
   id              SERIAL PRIMARY KEY,
   club_id         INTEGER REFERENCES clubs(id),
   name            VARCHAR(200) NOT NULL,
   category        VARCHAR(50), -- under 10, under 12, prima squadra, etc.
   season          VARCHAR(20), -- 2025-26

   -- Coach
   coach_id        INTEGER REFERENCES users(id),
   assistant_coach_id INTEGER REFERENCES users(id),

   -- Settings
   description     TEXT,
   logo_url        VARCHAR(500),

   is_active       BOOLEAN DEFAULT true,
   created_at      TIMESTAMP DEFAULT NOW()
);

-- ============================================
-- TEAM_MEMBERS (Relazione utente-squadra)
-- ============================================
CREATE TABLE team_members (
   id              SERIAL PRIMARY KEY,
   team_id         INTEGER REFERENCES teams(id),
   user_id         INTEGER REFERENCES users(id),

   role            VARCHAR(50) DEFAULT 'player', -- player, captain, coach
   jersey_number   INTEGER,
   position        VARCHAR(50),

   joined_at       TIMESTAMP DEFAULT NOW(),
   left_at         TIMESTAMP,

   UNIQUE(team_id, user_id)
);

5.2 Tabelle Eventi e Partite

-- ============================================
-- EVENTS
-- ============================================
CREATE TABLE events (
   id              SERIAL PRIMARY KEY,
   club_id         INTEGER REFERENCES clubs(id),
   team_id         INTEGER REFERENCES teams(id),

   -- Type
   event_type      VARCHAR(50), -- match, training, meeting, social
   title           VARCHAR(200),
   description     TEXT,

   -- Date/Time
   start_datetime  TIMESTAMP NOT NULL,
   end_datetime    TIMESTAMP,
   timezone        VARCHAR(50) DEFAULT 'Europe/Rome',

   -- Location
   venue_name      VARCHAR(200),
   venue_address   TEXT,
   latitude        DECIMAL(10,8),
   longitude       DECIMAL(11,8),

   -- RSVP
   rsvp_required   BOOLEAN DEFAULT true,
   rsvp_deadline   TIMESTAMP,

   -- Status
   status          VARCHAR(20) DEFAULT 'scheduled', -- scheduled, cancelled, completed

   created_by      INTEGER REFERENCES users(id),
   created_at      TIMESTAMP DEFAULT NOW()
);

-- ============================================
-- EVENT_ATTENDANCE (RSVP)
-- ============================================
CREATE TABLE event_attendance (
   id              SERIAL PRIMARY KEY,
   event_id        INTEGER REFERENCES events(id),
   user_id         INTEGER REFERENCES users(id),

   status          VARCHAR(20), -- attending, not_attending, maybe, no_response
   notes           TEXT,

   responded_at    TIMESTAMP,
   created_at      TIMESTAMP DEFAULT NOW(),

   UNIQUE(event_id, user_id)
);

-- ============================================
-- MATCHES (Partite)
-- ============================================
CREATE TABLE matches (
   id              SERIAL PRIMARY KEY,
   event_id        INTEGER REFERENCES events(id),

   -- Teams
   home_team_id    INTEGER REFERENCES teams(id),
   away_team_id    INTEGER REFERENCES teams(id),
   away_team_name  VARCHAR(200), -- per squadre esterne

   -- Competition
   competition     VARCHAR(200),
   matchday        INTEGER,

   -- Result
   home_score      INTEGER,
   away_score      INTEGER,
   result          VARCHAR(20), -- win, loss, draw, cancelled

   -- Status
   status          VARCHAR(20) DEFAULT 'scheduled', -- scheduled, live, finished, cancelled

   -- Scoring started by
   scorer_id       INTEGER REFERENCES users(id),

   created_at      TIMESTAMP DEFAULT NOW()
);

-- ============================================
-- MATCH_EVENTS (Eventi partita)
-- ============================================
CREATE TABLE match_events (
   id              SERIAL PRIMARY KEY,
   match_id        INTEGER REFERENCES matches(id),

   minute          INTEGER,
   event_type      VARCHAR(50), -- goal, card, substitution, timeout, etc.

   -- Players
   primary_player_id   INTEGER REFERENCES users(id),
   secondary_player_id INTEGER REFERENCES users(id), -- assist, replaced_by

   -- Details
   details         JSONB, -- sport-specific details

   created_at      TIMESTAMP DEFAULT NOW()
);

5.3 Tabelle Comunicazione

-- ============================================
-- NOTIFICATIONS
-- ============================================
CREATE TABLE notifications (
   id              SERIAL PRIMARY KEY,
   club_id         INTEGER REFERENCES clubs(id),

   -- Target
   user_id         INTEGER REFERENCES users(id), -- NULL = broadcast
   team_id         INTEGER REFERENCES teams(id), -- NULL = all teams

   -- Content
   title           VARCHAR(200),
   body            TEXT,
   type            VARCHAR(50), -- event_reminder, match_result, news, etc.

   -- Data
   data            JSONB, -- extra data for deep linking

   -- Status
   sent_at         TIMESTAMP,
   read_at         TIMESTAMP,

   created_at      TIMESTAMP DEFAULT NOW()
);

-- ============================================
-- MESSAGES (Chat)
-- ============================================
CREATE TABLE messages (
   id              SERIAL PRIMARY KEY,
   club_id         INTEGER REFERENCES clubs(id),
   team_id         INTEGER REFERENCES teams(id),

   sender_id       INTEGER REFERENCES users(id),

   -- Content
   content         TEXT,
   message_type    VARCHAR(20) DEFAULT 'text', -- text, image, file
   attachment_url  VARCHAR(500),

   -- Threading
   parent_id       INTEGER REFERENCES messages(id),

   created_at      TIMESTAMP DEFAULT NOW()
);

-- ============================================
-- NEWS/ANNOUNCEMENTS
-- ============================================
CREATE TABLE news (
   id              SERIAL PRIMARY KEY,
   club_id         INTEGER REFERENCES clubs(id),

   title           VARCHAR(200),
   content         TEXT,
   excerpt         TEXT,

   -- Media
   image_url       VARCHAR(500),

   -- Author
   author_id       INTEGER REFERENCES users(id),

   -- Status
   is_published    BOOLEAN DEFAULT false,
   published_at    TIMESTAMP,

   -- Engagement
   views_count     INTEGER DEFAULT 0,

   created_at      TIMESTAMP DEFAULT NOW()
);

-- ============================================

   id              SERIAL PRIMARY KEY,
   club_id         INTEGER REFERENCES clubs(id),

   name            VARCHAR(200),
   description     TEXT,
   website         VARCHAR(500),
   logo_url        VARCHAR(500),

   -- Tier
   tier            VARCHAR(50), -- main, gold, silver, bronze

   -- Contact
   contact_name    VARCHAR(200),
   contact_email   VARCHAR(200),

   view_type       VARCHAR(50), -- impression, click
   viewed_at       TIMESTAMP DEFAULT NOW()
);

-- ============================================
-- MEDIA (Foto/Video)
-- ============================================
CREATE TABLE media (
   id              SERIAL PRIMARY KEY,
   club_id         INTEGER REFERENCES clubs(id),
   team_id         INTEGER REFERENCES teams(id),
   event_id        INTEGER REFERENCES events(id),

   -- Uploader
   uploaded_by     INTEGER REFERENCES users(id),

   -- File
   file_url        VARCHAR(500),
   thumbnail_url   VARCHAR(500),
   file_type       VARCHAR(20), -- image, video

   title           VARCHAR(200),
   description     TEXT,

   -- Tags
   tags            TEXT[],
   tagged_users    INTEGER[],

   -- Status
   is_public       BOOLEAN DEFAULT true,

   created_at      TIMESTAMP DEFAULT NOW()
);

-- ============================================
-- PAYMENTS
-- ============================================
CREATE TABLE payments (
   id              SERIAL PRIMARY KEY,
   club_id         INTEGER REFERENCES clubs(id),
   user_id         INTEGER REFERENCES users(id),

   -- Payment details
   amount          DECIMAL(10,2),
   currency        VARCHAR(3) DEFAULT 'EUR',

   -- Type
   payment_type    VARCHAR(50), -- membership, match_fee, donation, merchandise
   description     TEXT,

   -- Status
   status          VARCHAR(20) DEFAULT 'pending', -- pending, completed, failed, refunded

   -- External
   external_id     VARCHAR(200), -- Stripe/PayPal ID

   paid_at         TIMESTAMP,
   created_at      TIMESTAMP DEFAULT NOW()
);

6. ARCHITETTURA TECNICA CONSIGLIATA
6.1 Stack Tecnologico

BACKEND:
• Node.js + Express / NestJS
• PostgreSQL (database principale)
• Redis (caching, sessioni)
• Socket.io (real-time updates)
• AWS S3 / Cloudinary (storage media)

FRONTEND WEB (Admin Dashboard):
• React 18+
• Tailwind CSS
• shadcn/ui components
• React Query (data fetching)
• Zustand (state management)

MOBILE APP:
• React Native (iOS + Android)
• Expo (sviluppo rapido)
• Push notifications (Firebase/OneSignal)

INFRASTRUTTURA:
• Docker (containerization)
• Railway / Render (hosting backend)
• Vercel (hosting frontend)
• AWS / DigitalOcean (storage)

6.2 Real-Time Architecture

LIVE SCORING E NOTIFICHE:

1. WEBSOCKET CONNECTION:
  • Socket.io per comunicazione bidirezionale
  • Rooms per club/team/match
  • Reconnection automatico

2. LIVE SCORING FLOW:
  Scorer App -> WebSocket -> Server -> Broadcast -> Fan Apps

3. NOTIFICATION SYSTEM:
  • Push notifications (Firebase Cloud Messaging)
  • In-app notifications
  • Email notifications (SendGrid/Mailgun)
  • SMS (Twilio) - opzionale

4. EVENT DRIVEN:
  • Redis Pub/Sub per eventi interni
  • Queue system per job asincroni (Bull/Agenda)

6.3 Multi-Tenant Architecture

OGNI CLUB HA:
• Database separato (schema isolation)
• Configurazione personalizzata
• Branding (colori, logo)
• Sottodominio personalizzato (clubname.platform.com)

VANTAGGI:
• Isolamento dati
• Scalabilità indipendente
• Backup separati
• Customizzazione completa
